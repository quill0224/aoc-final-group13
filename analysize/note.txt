Step 1: Load VGG.pth
- Parse Conv layer shapes
- Count weight sparsity per layer
- Get INT8 weight size

Step 2: Dense analytical model
- MACs
- activation / weight / psum bytes
- compute cycles
- memory cycles

Step 3: Sparse analytical model
- bitmask size
- compressed value size
- metadata overhead
- estimated PE utilization
- decode / routing overhead

Step 4: GLB sweep
- 8KB / 16KB / 32KB / 64KB
- check which tile can fit
- estimate DRAM traffic and total cycles
