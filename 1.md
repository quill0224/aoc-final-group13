# Trapezoid-Lite ASIC 系統架構與模組 I/O 介面規格書

此文件定義了基於 Trapezoid 演算法復刻的硬體目錄結構，以及三大核心模組 (Controller, GLB, DMA) 的完整 Port 列表。本版規格已徹底移除 Dense 矩陣專用的定址與 Padding 邏輯，全線介面對齊 TrIP 稀疏流 (Data 128-bit + Bitmask 16-bit) 與平坦化 Fiber 處理機制。

## 2. ASIC Controller 完整 Port 列表 (`top_controller.sv`)

負責接收軟體 CSR，管理適配 Trapezoid 演算法的 16-State 平坦化 FSM，驅動 DMA 與 MC 進行變長 Fiber 資料派發，管理 Scan Chain，並接收 PE 陣列、MFIU 與 PPU 的管線同步回報。

### 2.1 系統與 MMIO 參數配置 (From AXI Slave)
| Port 屬性 | 寬度 | 方向 | 名稱 | 功能描述 |
| :--- | :--- | :--- | :--- | :--- |
| `clk`, `rst` | 1 | `input` | System Clock & Reset | 系統時脈與高電位重置 |
| `asic_en` | 1 | `input` | ASIC Enable | 啟動運算 (觸發 IDLE $\to$ LOAD_A) |
| `asic_done` | 1 | `output` | ASIC Done | 運算完成中斷 (DONE 狀態拉高) |
| `operation_mode` | 2 | `input` | Operation Mode | 運算模式: `00`=Dense(IP), `01`=TrIP, `10`=TrGS |
| `A_fiber_base_addr`| 32 | `input` | A Fiber Base Addr | A Fiber (原 IFMap) 於 DRAM 之實體基底位址 |
| `B_fiber_base_addr`| 32 | `input` | B Fiber Base Addr | B Fiber (原 Filter) 於 DRAM 之實體基底位址 |
| `C_tensor_base_addr`|32 | `input` | C Tensor Base Addr | 輸出特徵圖寫回 DRAM 之實體基底位址 |
| `comp_A_len` | 32 | `input` | Compressed A Length | 軟體指定的當前 A Fiber 壓縮長度 (位址累加用) |
| `comp_B_len` | 32 | `input` | Compressed B Length | 軟體指定的當前 B Fiber 壓縮長度 (位址累加用) |
| `tile_total_count` | 16 | `input` | Tile Total Count | 全層總 Tile 數 (供 UPDATE 狀態檢查結束邊界) |
| `m`, `e`, `p`, `q`, `r`, `t` | 10/4/3 | `input` | Mapping Parameters | 底層硬體空間折疊映射參數 |
| `GLB_A_base_addr` | 16 | `input` | GLB A Base Addr | A Fiber 於 GLB 內部之基底位址 |
| `GLB_B_base_addr` | 16 | `input` | GLB B Base Addr | B Fiber 於 GLB 內部之基底位址 |
| `GLB_C_base_addr` | 16 | `input` | GLB C Base Addr | 運算輸出寫回 GLB 內部之基底位址 |

### 2.2 對接 DMA Engine (搬運控制)
| Port 屬性 | 寬度 | 方向 | 名稱 | 功能描述 |
| :--- | :--- | :--- | :--- | :--- |
| `DMA_en` | 1 | `output` | DMA Enable | 觸發 DMA 啟動動作 (LOAD_A / LOAD_B / SEND_OFMAP) |
| `DMA_mode` | 2 | `output` | DMA Mode | 指示目標: `00`=A, `01`=B, `10`=保留, `11`=C Tensor WB |
| `DMA_DRAM_ADDR` | 32 | `output` | DMA DRAM Address | 來源/目的 DRAM 實體位址 (由內部 Address Accumulator 提供) |
| `DMA_GLB_ADDR` | 16 | `output` | DMA GLB Address | 對應寫入或讀出的 GLB 內部位址 |
| `DMA_len` | 32 | `output` | DMA Length | 傳輸的字節長度 (直連 `comp_A_len` / `comp_B_len`) |
| `DMA_done` | 1 | `input` | DMA Done | 該次 DMA 任務完成回報 (觸發 FSM 狀態轉移) |

### 2.3 對接 Memory Controller (派發與交握)
| Port 屬性 | 寬度 | 方向 | 名稱 | 功能描述 |
| :--- | :--- | :--- | :--- | :--- |

| `mc_B_done` | 1 | `input` | MC B Done | B Fiber 派發完畢 (觸發 FSM 進入 MFIU_INTERSECT) |

### 2.4 對接 PE Array & MFIU (運算控制與同步)
| Port 屬性 | 寬度 | 方向 | 名稱 | 功能描述 |
| :--- | :--- | :--- | :--- | :--- |

| `PEA_B_ready` | 1 | `input` | PEA B Ready | B-Network 全體 PE FIFO 之 AND Ready 訊號 (木桶效應) |
| `mfiu_done` | 1 | `input` | MFIU Done | PE 內 MFIU 交集與前綴和組合邏輯穩定訊號 |
| `distribute_done`| 1 | `input` | Distribute Done | Benes Network 路由繞線穩定訊號 |

### 2.5 掃描鏈介面 (Scan Chain - 保留供佈署)

| :--- | :--- | :--- | :--- | :--- |

| `LN_config_in` | M | `output` | LN Config In | PE 內部 Local Network 路由設定輸入 |

### 2.6 對接 GON & PPU (寫回與後處理)
| Port 屬性 | 寬度 | 方向 | 名稱 | 功能描述 |
| :--- | :--- | :--- | :--- | :--- |

## 3. Global Buffer Port 列表 (`global_buffer.sv`)

全域內部記憶體。為適配 TrIP 架構，資料線寬度已由 128-bit 擴增為 144-bit (128-bit Data + 16-bit Bitmask)。

| Port 屬性 | 寬度 | 方向 | 名稱 | 功能描述 |
| :--- | :--- | :--- | :--- | :--- |
| `clk`, `rst` | 1 | `input` | Clock / Reset | 系統時脈與重置 |
| `glb_en` | 1 | `input` | GLB Enable | SRAM 致能信號 (Active Low/High 依實體 IP 規格) |
| `glb_web` | 1 | `input` | GLB Write Enable | 寫入致能 (0=Write, 1=Read) |
| `glb_mode` | 1 | `input` | GLB Mode | 存取模式 (`WORD_MODE` / `BYTE_MODE`) |
| `glb_addr` | 16 | `input` | GLB Address | SRAM 讀寫實體位址 |
| `glb_data_in` | 144| `input` | GLB Data Input | 寫入資料 (DMA 搬入 Fiber，或 PPU 寫入 C Tensor) |
| `glb_data_out` | 144| `output`| GLB Data Output| 讀出資料 (供給 MC 解析 Bitmask 並派發，或 DMA 回寫) |

## 4. DMA Engine Port 列表 (`DMA.sv`)

負責將 AXI4 介面之外部 DRAM 變長封包 (Fiber) 與內部 GLB 進行高速對接。本模組不包含位址產生邏輯，絕對位址與長度由 Controller 直接下達。

### 4.1 控制介面 (對接 Controller)
| Port 屬性 | 寬度 | 方向 | 名稱 | 功能描述 |
| :--- | :--- | :--- | :--- | :--- |
| `clk`, `rst` | 1 | `input` | Clock / Reset | 系統時脈與重置 |
| `EN` | 1 | `input` | Enable | 觸發 DMA 啟動 |
| `MODE` | 2 | `input` | Target Mode | A(00), B(01), Bias(10), C_Tensor(11) |
| `DRAM_ADDR` | 32 | `input` | DRAM Address | 外部記憶體之基底/偏移位址 |
| `GLB_ADDR` | 16 | `input` | GLB Address | 內部 SRAM 之目標位址 |
| `LEN` | 32 | `input` | Length | 該次任務搬運總長度 |
| `DONE` | 1 | `output`| Done | 該次搬運完成回報 |

### 4.2 AXI4 Master 介面 (對接 SoC/DRAM)

// ==============================================================================
// 模組：ASIC Controller (Trapezoid 16-State 平坦化架構)
// ==============================================================================
module top_controller (
    // 系統與控制訊號
    input  logic                                clk,
    input  logic                                rst,
    input  logic                                asic_en,
    output logic                                asic_done,

    // MMIO CSR 配置 (來自 AXI Slave)
    input  logic [1:0]                          operation_mode,     // 00:Dense(IP), 01:TrIP, 10:TrGS
    input  logic [`AXI_ADDR_BITS-1:0]           A_fiber_base_addr,  // A Fiber 於 DRAM 之實體基底位址
    input  logic [`AXI_ADDR_BITS-1:0]           B_fiber_base_addr,  // B Fiber 於 DRAM 之實體基底位址
    input  logic [`AXI_ADDR_BITS-1:0]           C_tensor_base_addr, // Output 寫回 DRAM 之實體基底位址
    
    input  logic [`GLB_ADDR_BITS-1:0]           GLB_A_base_addr,    // A Fiber 於 GLB 內部之基底位址
    input  logic [`GLB_ADDR_BITS-1:0]           GLB_B_base_addr,    // B Fiber 於 GLB 內部之基底位址
    input  logic [`GLB_ADDR_BITS-1:0]           GLB_C_base_addr,    // Output 寫回 GLB 內部之基底位址

    input  logic [31:0]                         comp_A_len,         // 軟體配置之當前 A Fiber 壓縮總字節長度
    input  logic [31:0]                         comp_B_len,         // 軟體配置之當前 B Fiber 壓縮總字節長度
    input  logic [15:0]                         tile_total_count,   // 全層總 Tile 數 (供 UPDATE 檢查結束邊界)

    // 網路特徵形狀參數 (Shape Parameters)
    input  logic [9:0]                          C,
    input  logic [9:0]                          M,
    input  logic [7:0]                          W,
    input  logic [7:0]                          H,

    // 硬體空間映射參數 (Mapping Parameters)
    input  logic [9:0]                          m,
    input  logic [3:0]                          e,
    input  logic [2:0]                          p,
    input  logic [2:0]                          q,
    input  logic [2:0]                          r,
    input  logic [2:0]                          t,

    // DMA 搬運介面
    output logic                                DMA_en,             // 觸發 DMA 啟動
    output logic [1:0]                          DMA_mode,           // 00:A, 01:B, 10:保留, 11:C Tensor WB
    output logic [`AXI_ADDR_BITS-1:0]           DMA_DRAM_ADDR,      // DRAM 存取位址 (由內部 Address Accumulator 產出)
    output logic [`GLB_ADDR_BITS-1:0]           DMA_GLB_ADDR,       // GLB 存取位址
    output logic [31:0]                         DMA_len,            // 當次搬運長度
    input  logic                                DMA_done,           // DMA 完成中斷

    // Memory Controller 派發介面
    output logic                                mc_start_A,         // 觸發 MC 推送 A Fiber (單播)
    output logic                                mc_start_B,         // 觸發 MC 推送 B Fiber (廣播)
    input  logic                                mc_A_done,          // A Fiber 派發完畢
    input  logic                                mc_B_done,          // B Fiber 派發完畢

    // PE Array & MFIU 運算控制與交握
    output logic [1:0]                          global_mode,        // 廣播當前運算模式至全體 PE
    output logic                                global_flush,       // COMPUTE 結束，解鎖 PE Local Buffer
    output logic [`PE_ARRAY_H*`PE_ARRAY_W-1:0]  PE_en,              // PE 致能遮罩
    output logic [10:0]                         PE_config,          // PE 組態參數
    
    input  logic                                PEA_A_ready,        // A-Network 目標 PE FIFO Ready
    input  logic                                PEA_B_ready,        // B-Network 全體 PE FIFO Ready (AND 邏輯)
    input  logic                                mfiu_done,          // MFIU 交集運算與 Prefix Sum 穩定訊號
    input  logic                                distribute_done,    // Benes Network 路由穩定訊號

    // 掃描鏈配置介面 (保留供佈署)
    output logic                                set_XID,
    output logic                                set_YID,
    output logic                                set_LN,
    output logic [`XID_BITS-1:0]                ifmap_XID_scan_in,
    output logic [`XID_BITS-1:0]                filter_XID_scan_in,
    output logic [`XID_BITS-1:0]                ipsum_XID_scan_in,
    output logic [`XID_BITS-1:0]                opsum_XID_scan_in,
    output logic [`YID_BITS-1:0]                ifmap_YID_scan_in,
    output logic [`YID_BITS-1:0]                filter_YID_scan_in,
    output logic [`YID_BITS-1:0]                ipsum_YID_scan_in,
    output logic [`YID_BITS-1:0]                opsum_YID_scan_in,
    output logic [`PE_ARRAY_H-2:0]              LN_config_in,

    // GON & PPU 寫回與後處理介面
    input  logic                                PEA_opsum_valid,    // PE Row 完成 K 維度累加，發出寫回請求
    output logic                                PEA_opsum_ready,    // Controller (或 Arbiter) 許可該 Row 寫回
    output logic [`XID_BITS-1:0]                opsum_tag_X,        // 寫回結果之空間座標標籤 X
    output logic [`YID_BITS-1:0]                opsum_tag_Y,        // 寫回結果之空間座標標籤 Y

    output logic                                Maxpool_en,         // PPU 池化致能
    output logic                                Maxpool_init,       // PPU 池化暫存器初始化
    output logic                                relu_sel,           // PPU ReLU 激活選擇
    input  logic                                ppu_done            // PPU INT8 寫回 GLB 完成訊號
);

// 模組：Global Buffer (支援 TrIP 144-bit 資料寬度)
// ==============================================================================
module global_buffer (
    input  logic                                clk,
    input  logic                                rst,

    // SRAM 控制介面
    input  logic                                glb_en,       // SRAM 致能 (Active Low/High 視實體 Memory Compiler 而定)
    input  logic                                glb_web,      // 寫入致能 (0: Write, 1: Read)
    input  logic                                glb_mode,     // 存取模式 (例如 0: Byte, 1: Word)
    input  logic [`GLB_ADDR_BITS-1:0]           glb_addr,     // 內部實體位址 (16-bit, 支援 64KB 定址)

    // 資料讀寫介面 (144-bit = 128-bit Data + 16-bit Bitmask)
    input  logic [143:0]                        glb_data_in,  // 寫入資料 (來自 DMA 或 PPU)
    output logic [143:0]                        glb_data_out  // 讀出資料 (供給 DMA 寫回，或 MC 派發)
);

// ==============================================================================
// 模組：DMA Engine (變長 Fiber 搬運)
// ==============================================================================
module dma_engine (
    input  logic                                clk,
    input  logic                                rst,

    // 控制器對接介面
    input  logic                                EN,             // 啟動 DMA 搬運
    input  logic [1:0]                          MODE,           // 搬運模式 (00:A, 01:B, 10:保留, 11:C Tensor WB)
    input  logic [`AXI_ADDR_BITS-1:0]           DRAM_ADDR,      // 來源/目的 DRAM 實體位址
    input  logic [`GLB_ADDR_BITS-1:0]           GLB_ADDR,       // 來源/目的 GLB 內部偏移位址
    input  logic [31:0]                         LEN,            // 傳輸總長度
    output logic                                DONE,           // 搬運任務完成訊號

    // GLB 存取介面 (對接 GLB Mux)
    output logic                                GLB_EN_dma,     // DMA 要求存取 GLB
    output logic                                GLB_WEB_dma,    // DMA 寫入/讀出 GLB 控制 (0: Write, 1: Read)
    output logic                                GLB_MODE_dma,   // 存取單位模式
    output logic [`GLB_ADDR_BITS-1:0]           GLB_A_dma,      // DMA 內部計數器產生的 GLB 連續位址
    output logic [143:0]                        GLB_DI_dma,     // DMA 寫入 GLB 的資料 (DRAM -> GLB)
    input  logic [143:0]                        GLB_DO_dma,     // GLB 讀出給 DMA 的資料 (GLB -> DRAM)

    // AXI4 Master 介面 (To DRAM / SoC)
    output logic [`AXI_ID_BITS-1:0]             AWID_M,
    output logic [`AXI_ADDR_BITS-1:0]           AWADDR_M,
    output logic [`AXI_LEN_BITS-1:0]            AWLEN_M,
    output logic [`AXI_SIZE_BITS-1:0]           AWSIZE_M,
    output logic [1:0]                          AWBURST_M,
    output logic                                AWVALID_M,
    input  logic                                AWREADY_M,

    output logic [`AXI_DATA_BITS-1:0]           WDATA_M,
    output logic [`AXI_STRB_BITS-1:0]           WSTRB_M,
    output logic                                WLAST_M,
    output logic                                WVALID_M,
    input  logic                                WREADY_M,

    input  logic [`AXI_ID_BITS-1:0]             BID_M,
    input  logic [1:0]                          BRESP_M,
    input  logic                                BVALID_M,
    output logic                                BREADY_M,

    output logic [`AXI_ID_BITS-1:0]             ARID_M,
    output logic [`AXI_ADDR_BITS-1:0]           ARADDR_M,
    output logic [`AXI_LEN_BITS-1:0]            ARLEN_M,
    output logic [`AXI_SIZE_BITS-1:0]           ARSIZE_M,
    output logic [1:0]                          ARBURST_M,
    output logic                                ARVALID_M,
    input  logic                                ARREADY_M,

    input  logic [`AXI_ID_BITS-1:0]             RID_M,
    input  logic [`AXI_DATA_BITS-1:0]           RDATA_M,
    input  logic [1:0]                          RRESP_M,
    input  logic                                RLAST_M,
    input  logic                                RVALID_M,
    output logic                                RREADY_M
);
